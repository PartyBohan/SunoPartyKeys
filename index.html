⚙️ 设置

MIDI: … — — —
Suno API 连接
代理模式：跑随附的 local-proxy.mjs（node local-proxy.mjs sk_live_你的key），或把 api/suno.js 部署到 Vercel。代理藏住密钥 + 解决 CORS。生成失败会自动降级为内置伴奏。
分轨
分轨走第 3 步。代理模式：Vercel 加环境变量 MUSICAI_KEY（部署 api/stems.js），或本地 MUSICAI_KEY=... node local-proxy.mjs sk_live_...。在 music.ai 后台建一个人声/伴奏分离 workflow，把它的 slug 填上面。没填或 demo 模式则跳过分轨，跟弹用整首伴奏。
分轨走第 3 步。本地跑随附的 demucs-server.py（pip install -U demucs + ffmpeg，再 python3 demucs-server.py）。它把 Suno 整曲拆成人声 + 伴奏两轨，纯本地、无密钥、CPU 约 30–90 秒。没起服务则降级用整首伴奏。
分轨走第 3 步。云端跑 Replicate 上的 Demucs（和本地同一个 htdemucs）。只需在 Vercel 加环境变量 REPLICATE_API_TOKEN（部署 api/stems-replicate.js）。按秒计费、个人可用，一首约几分钱。无需填任何 slug 或 key 到页面里。没配 token 或 demo 模式则跳过分轨，跟弹用整首伴奏。

🛠️ 研发历程 · Development journey
从 MVP 到「光音成曲」的迭代记录 · From MVP to Shine Up, Notes On
v0
搭建 MVP 流程
完成首个三步向导：录制 → 调音 → 跟弹，配 36 键钢琴与基础设置。
v1
确立视觉风格
加入蓝–紫–橙渐变、毛玻璃界面、动态背景、中英双语与 PartyKeys MIDI 检测。
v2
简化音乐描述
做了标签式提示词构建器，用户选风格、情绪、乐器和速度，无需手写提示词。
v3
解决混音难题
决定保留 Suno 生成，另加分轨步骤，把旋律和背景音乐分开。
v3b
增加分轨后端
提供两条分轨路径：本地 Demucs 离线/演示用，Music.ai 云端用，失败回退整首混音。
v4
重构产品流程
改成完整四步闭环：录制 → 调音 → 分轨 → 跟弹，并加入分轨混音台。
v5
音频转可弹音符
做了人声转 MIDI 检测，把分出的旋律变成瀑布流所需的音高与时值。
v5b
提升识谱准确度
修复假音符、过度切分、音高偏移和八度错误，让识别出的旋律更干净。
v6
搭建灯光教学
加入瀑布流跟弹系统：判定线、连击、进度、准确度评分和 LED 反馈。
v6b
优化性能
用 GPU 加速的 CSS 动画重写瀑布流，跟弹更流畅、同步更准。
v7
加强新手引导
加入 0.3 秒预亮、两段式琴键发光、硬件 LED 同步与命中迸发反馈。
v8
打磨布局
放大主卡片与瀑布流，键盘全宽显示，并改进琴键的响应式定位。
v9
优化生成时刻
主按钮改名「SUNO – Generate!」，生成曲播放时加入跑道式光环。
v10
升级跟弹体验
加入跟弹模式：弹错时音乐暂停，直到弹对才继续；并加强庆祝特效。
v11
增加学习模式
加入 跟弹 / 连弹 切换：跟弹等你弹对，连弹保持音乐前进；并平滑八度跳跃、去掉过密音符。
v12
改善等待体验
分轨进行时先试听预览，分轨完成或失败后淡出。
v13
确定品牌
产品定名 Shine Up, Notes On（首字母藏 SUNO），并加上中文名「光音成曲」。
